# Code of Conduct

## Our Standard

We want Catalyst Studio to be a useful, respectful, and technically rigorous
open-source project. Contributors and maintainers are expected to:

- Use welcoming and inclusive language.
- Respect differing viewpoints and experience levels.
- Give and receive constructive technical feedback.
- Keep discussions focused on the project and its users.
- Avoid harassment, personal attacks, and discriminatory language.

## Unacceptable Behavior

Unacceptable behavior includes:

- Harassment, intimidation, or sustained disruption.
- Insults, slurs, or sexualized language or imagery.
- Publishing private information without explicit permission.
- Trolling, personal attacks, or bad-faith arguments.
- Any conduct that would be inappropriate in a professional setting.

## Enforcement

Project maintainers may remove comments, close issues or pull requests, or
restrict participation when behavior violates this code of conduct.

Report concerns privately using the security or maintainer contact listed in
`SECURITY.md`. Maintainers will review reports promptly and handle them with
as much confidentiality as practical.
